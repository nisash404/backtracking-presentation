Backtracking

  • How do algorithms solve puzzles that seem impossible?
  • The secret: knowing when to change your mind
  • No panicking — just systematic trial and error
02

What is Recursion?

  • A function that calls itself to solve smaller subproblems
  • It only moves forward — deeper and deeper
  • If it makes a wrong choice, it has no way to undo it

⚙️ The Base Case

  • The condition that tells recursion "you can stop now"
  • Like the smallest nesting doll — nothing left to open
  • Without it, recursion never stops

💥 Stack Overflow

  • Without a base case, the function calls itself infinitely
  • Each call stacks a new frame in memory
  • Eventually the stack runs out of space — crash 💥
03

Enter: Backtracking

  • Recursion's problem: it can't undo a bad decision
  • Backtracking adds a rewind button to recursion
  • If a path fails, go back and try the next one

�️ The Time Traveler

  • Imagine being blindfolded in a maze and hitting a dead end
  • Instead of restarting, you rewind to your last intersection
  • Then calmly try the other path — that's backtracking
04

Backtracking on a Tree

  • The algorithm starts at the root and explores each branch depth-first
  • If a node isn't the target, it backtracks to the parent and tries the next child
  • Dead-end branches are abandoned — only the solution path survives

How to Cheat: Pruning

  • Without optimization, backtracking tries every possible path
  • That's way too slow — the trick is called Pruning
  • We skip branches we already know will fail

🚧 The "Bridge Out" Rule

  • If a sign says "Bridge Out Ahead", you don't drive to verify
  • You turn around immediately and save time
  • Same idea — don't explore paths that are obviously wrong

✂️ Constraint Checking

  • Before each step, the algorithm asks: "Is this valid?"
  • If it's a wall or already visited, it skips that path entirely
  • This prevents wasted work and avoids stack overflow

The Boss Fight: 4×4 Queens

  • Goal: place 4 queens so no two can attack each other
  • If a queen causes a conflict, the algorithm removes her and tries the next column
  • A classic problem that only backtracking can solve efficiently

The 3-Step Formula

  • Every backtracking algorithm follows the same 3-step pattern
  • Choose a move, Explore it recursively, then Un-choose (undo) it
  • This simple loop solves mazes, queens, Sudoku — everything
def solve(path, choices):
    if is_goal(path): # THE BRAKES (Base Case)
        return print(path)
 
    for move in choices:
        if is_valid(move): # PRUNING (Bridge Out)
 
            path.append(move) ← CHOOSE
            solve(path, choices) ← EXPLORE
            path.pop() ← UN-CHOOSE

Real World Applications

  • Backtracking isn't just theory — these apps use it every day
🧩

Sudoku

  • Places a number, checks the rules — if a conflict appears later, it rewinds
  • Removes that number and tries the next one
🗺️

Maps & Routing

  • When GPS hits a closed road, it backtracks to find an alternate route
📅

University Timetables

  • Assigning a class to a slot that causes a conflict? Undo it and try another
🤖

Game AI

  • Chess engines explore moves ahead — if a move leads to losing, they rewind and try a better one
